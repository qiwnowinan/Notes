# 记录python中的常用模块

## 模块

简单来说，任何一个以 .py 为后缀的 Python 文件，就是一个模块。里面可以存放函数、类、变量等代码，以便在其他程序中方便地使用，从而避免重复编写相同的代码。

| 导入方式 | 核心语法 | 特点与说明 |
| :--- | :--- | :--- |
| 基础导入 | `import 模块名` | 导入整个模块，使用时需加 `模块名.` 前缀。 |
| 别名导入 | `import 模块名 as 别名` | 给模块起一个简短的别名，常用于模块名较长时。 |
| 精准导入 | `from 模块名 import 成员` | 只导入模块中特定的函数或变量，可直接使用，无需前缀。 |

模块和包 (Package)  
模块和包共同构成了 Python 的代码组织结构，它们的关系如下：
模块 (Module)：一个 .py 文件，是代码组织的基本单元。
包 (Package)：一个包含 __init__.py 文件的目录，用于组织和管理多个相关的模块。
简单来说，包是模块的集合。例如，numpy 就是一个非常流行的第三方包，它内部包含了数十个用于科学计算的模块。

## range

它不是列表,你要一个，它给你一个，而不是一次性把所有东西都生产出来堆在地上,常用于for循环  
`range(start, stop, step)`  
stop（结束值）—— 必填,  
start（起始值）—— 选填（默认为 0）,  
step（步长）—— 选填（默认为 1）  

## random

`random.random()`  
生成一个 [0.0, 1.0) 之间的随机浮点数。包含 0.0，但不包含 1.0。常用于概率判断。
`if random.random() < 0.3: # 有 30% 的概率执行某事`

`random.uniform(a, b)`  
生成一个 [a, b] 之间的随机浮点数,通常包含边界值 a 和 b。

`random.randint(a, b)`  
生成一个 [a, b] 之间的随机整数。

`random.randrange(start, stop[, step])`  
从指定范围、按指定步长递增的集合中获取一个随机整数,不包含 stop,用于生成特定规律的随机数。

## Path

适合小文件，处理文件路径  
先`from pathlib import Path`  

```python
file_path = Path("D:/工作") / "照片" / "猫.jpg" #路径表示

print(file_path) 
# 在 Windows 上它会自动变成: D:\工作\照片\猫.jpg
```

### 创建文件夹

```py
from pathlib import Path

# 1. 定义你想创建的文件夹路径
# 比如：在当前目录下创建 "data" 文件夹，里面再建一个 "2023" 子文件夹
folder_path = Path("data") / "2023"

# 2. 创建文件夹
# parents=True: 如果父文件夹（data）不存在，顺便一起创建
# exist_ok=True: 如果文件夹已经存在，不要报错，直接跳过
folder_path.mkdir(parents=True, exist_ok=True)

print(f"✅ 文件夹已就绪：{folder_path}")
```

### 获取信息

```py
from pathlib import Path
p = Path("D:/Work/Report_2023.pdf")
# 1. 只要文件名（带后缀）
print(p.name)       
# 输出: Report_2023.pdf

# 2. 只要文件名（不要后缀，这个最常用！）
print(p.stem)       
# 输出: Report_2023

# 3. 只要后缀
print(p.suffix)     
# 输出: .pdf

# 4. 它的父文件夹
print(p.parent)     
# 输出: D:\Work

# 5. 它的父文件夹的父文件夹
print(p.parent.parent) 
# 输出: D:\

# 如果想把上面的文件改名成 .txt，可以直接修改属性：(只是改了new_file变量而已)
new_file = p.with_suffix('.txt')
print(new_file) 
# 输出: D:\Work\Report_2023.txt
```

### 判断状态

```py
p = Path("test.txt")

if p.exists():        # 问：存在吗？
    if p.is_file():   # 问：是文件吗？

## 读取文本文件（with）

### open

`open(file, mode='r', buffering=-1, encoding=None, errors=None, newline=None, closefd=True, opener=None)`

```py
with open('example.txt', 'r', encoding='utf-8') as file:
    content = file.read()
    print(content)
# 文件在这里自动关闭
```

#### 参数

1. file (文件路径)
类型：字符串 (str) 或路径对象 (pathlib.Path)
作用：告诉 Python 文件在哪里，或者想创建的文件叫什么名字。
示例：'data/log.txt', 'C:/Users/test.csv'
2. mode (模式) —— 最重要
    类型：字符串
    作用：决定你是要读、要写，还是要处理图片。
    常用值：

    | 模式 | 读 | 写 | 指针位置 | 文件存在 | 文件不存在 | 备注 |
    | :--- | :---: | :---: | :--- | :--- | :--- | :--- |
    | `'r'` | ✅ | ❌ | 开头 | 打开 | 报错 | 默认模式 |
    | `'w'` | ❌ | ✅ | 开头 | 清空 | 创建 | 小心覆盖！ |
    | `'a'` | ❌ | ✅ | 末尾 | 打开 | 创建 | 安全追加 |
    | `'rb'` / `'wb'` | - | - | - | - | - | 处理图片/视频/pdf/zip |

3. encoding (编码)
类型：字符串
作用：指定文件的“翻译字典”。如果不指定，Windows 和 Mac/Linux 可能会用不同的默认编码，导致乱码。如果是二进制文件，不要写
常用值：
'utf-8'：通用标准。适用于绝大多数代码文件、Linux 服务器、Web 开发。
'utf-8-sig'：Excel 专用。专门用来生成 CSV 文件，防止 Excel 打开中文乱码。
4. newline (换行符控制)
类型：字符串
作用：控制换行符的处理方式。在使用 csv 模块写文件时，必须设为 ''。
常用值：
'' (空字符串)：禁用换行符转换。这是配合 csv 模块的标准写法，防止 Excel 中出现空行。
'\n'：强制使用 Linux 风格换行。
None (默认)：Python 会根据操作系统自动转换换行符（Windows 下 \n 变 \r\n）。

### f的方法

1. f.read(size) —— 全拿或分块拿  
用法：  
f.read()：不填数字，一口气读完。适合小文件。  
f.read(10)：填了数字，只读 10 个字符（文本模式）或 10 字节（二进制模式）。适合处理几个 G 的大文件，防止内存爆炸。  
返回类型：字符串（文本模式）或 字节流（二进制模式）。  
2. f.readline() —— 按行拿  
用法：f.readline()  
特点：  
每次只读一行（包括末尾的 \n 换行符）。  
适合内存不够大，但需要逐行分析数据的场景。  
3. f.readlines() —— 打包成列表拿  
这是最“偷懒”但也最占内存的方法。  
用法：f.readlines()  
特点：  
把文件的每一行都读出来，塞进一个列表里。  
返回结果像这样：['第一行\n', '第二行\n', '第三行']。  
缺点：如果文件有 10GB，电脑内存可能直接爆掉。  
4. f.write(string) —— 写入字符串  
用法：f.write("你好\n")  
关键点：  
不会自动换行！如果你想换行，必须手动加 \n。  
只能写字符串。如果你想写数字，得先 str(123) 转成字符串。  
返回值：返回写入的字符长度。  
5. f.writelines(lines) —— 写入列表  
当有一堆数据在列表里，想一次性存进去时用这个。  
用法：f.writelines(['第一行\n', '第二行\n'])  
关键点：  
它接受一个列表作为参数。  
也不会自动换行！列表里的每个字符串必须自己带好 \n，否则它们会连成一串。

## CSV

先要引入`import csv`

### 读

#### 用reader

```py
with open('example.csv', 'r', encoding='utf-8-sig', newline='') as f:
    csv_reader = csv.reader(f) # csv_reader是可迭代对象
    # 是一行一行地读取，而且是“读一行，给一行”
    
    for line in csv_reader: # 所以必须在with缩进下
        pass                # line是list
```

#### 用DictReader

```py
import csv

with open('data.csv', 'r', encoding='utf-8-sig', newline='') as f:
    csv_reader = csv.DictReader(f)
    
    for row in csv_reader: # 返回字典如： {'姓名': '张三', '年龄': '25'}
        print(f"{row['姓名']} 住在 {row['城市']}")
```

### 写

#### 用writer

```py
with open('example.csv', 'w', encoding='utf-8-sig', newline='') as f:
    csv_writer = csv.writer(f) 
    
    header = ['Name', 'Chinese', 'English']
    csv_writer.writerow(header)

    rows = [
        ['Apple', '苹果', 'apples'],
        ['Book', '书', 'book'],
        ['Cat', '猫', 'cat']
    ]
    csv_writer.writerows(rows)
```

#### 用DictWriter

```py
import csv

# 数据（列表套字典）
data = [
    {'name': 'Apple', 'cn': '苹果'}, 
    {'name': 'Book', 'cn': '书'}
]

# 表头（必须和字典的键对应）
headers = ['name', 'cn'] 

with open('data.csv', 'w', encoding='utf-8-sig', newline='') as f:
    writer = csv.DictWriter(f, fieldnames=headers)
    
    writer.writeheader()      # 写入表头
    writer.writerows(data)    # 写入数据
```

## JSON

JSON本质是字符串  
JSON 的核心结构只有两种：
对象：用花括号 {} 包裹，由“键值对”组成（类似于 Python 的字典）。  
数组：用方括号 [] 包裹，存放有序的列表（类似于 Python 的列表）。  

| 维度 | JSON | Python | 关键差异点 |
| :--- | :--- | :--- | :--- |
| 本质 | 文本字符串 | 内存对象 | JSON 是死的文本，Python 是活的对象 |
| 对象/字典 | `{ "key": "value" }` | `{ "key": "value" }` | JSON 键必须双引号；Python 键可是任意不可变类型 |
| 数组/列表 | `[ 1, 2, 3 ]` | `[ 1, 2, 3 ]` | 基本一致 |
| 布尔值 | `true` / `false` (小写) | `True` / `False` (大写) | 大小写不同 |
| 空值 | `null` | `None` | 单词不同 |
| 注释 | 不支持 | 支持 (`#`) | JSON 极其严格，不能有注释 |

```py
import json

data = {"name": "Alice", "age": 30, "is_student": False, "skills": ["Python", "Java"]}

# 序列化：转为字符串
json_str = json.dumps(data)
print(json_str)
print(type(json_str)) # <class 'str'>
```

### 方法

- json.dumps()  
它的作用是把 Python 里的字典或列表，变成一串 JSON 格式的文本。  
输入：Python 对象（字典、列表）。  
输出：JSON 字符串 (str)。  

```py
import json

data = {"name": "张三", "age": 25}

# 默认转换（中文会变成乱码编码）
s1 = json.dumps(data) 
# 结果: '{"name": "\u5f20\u4e09", "age": 25}'

# 推荐写法（加上参数）
s2 = json.dumps(data, ensure_ascii=False, indent=4)
# 结果: 一个漂亮的、包含中文的 JSON 字符串
```

- json.loads()  
当拿到一串 JSON 文本时，Python 没法直接用它，得把它“翻译”回 Python 的字典。  
输入：JSON 字符串 (str)。  
输出：Python 对象（通常是字典）。  

```py
import json

# 假设这是从网上拿到的字符串
json_text = '{"name": "李四", "age": 30, "city": "Beijing"}'

# 把它“加载”回内存
data = json.loads(json_text)
```

- json.dump()  
写入文件,这个方法先把 Python 对象转成 JSON，然后直接写进硬盘里的文件。  
输入：Python 对象 + 文件对象。  

```py
import json

data = {"product": "电脑", "price": 5000}

# 先打开一个文件（用 'w' 写入模式）
with open("data.json", "w", encoding="utf-8") as f:
    # 把数据 dump 进去
    json.dump(data, f, ensure_ascii=False, indent=4)
```

- json.load()  
读取文件，直接从硬盘读取 JSON 文件，并把它变成 Python 字典。  
输入：文件对象。  
输出：Python 对象（字典）。  

```py
import json

# 打开文件（用 'r' 读取模式）
with open("data.json", "r", encoding="utf-8") as f:
    # 直接从文件 load 数据
    data = json.load(f)

print(data) # 输出: {'product': '电脑', 'price': 5000}
