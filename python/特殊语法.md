## match-case

match-case 是 Python 3.10 引入的“超级 switch”，专门用来处理复杂的逻辑分支。它不仅能像传统的 switch 语句那样匹配具体的值，还能直接拆解列表、字典等数据结构，不需要break  

### 普通用法  

```python
def get_status_code(status):
    match status:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case 500:
            return "Server Error"
        case _:  # 通配符，相当于 default，匹配任何剩余情况,必须放在最后
            return "Unknown Status"

print(get_status_code(404))  # 输出: Not Found
```

### 多模式匹配  

```python
status = 403
match status:
    case 400 | 401 | 403 | 404:
        print("客户端错误")
    case 500:
        print("服务器错误")
```

### 结构模式匹配  

#### 匹配列表/元组  

```python
point = [10, 20]

match point:
    case [0, 0]:
        print("原点")
    case [0, y]:
        print(f"在Y轴上，Y坐标是 {y}")  # 自动提取了 y = 20  列表里第一个必须是 0，第二个可以是任意值（赋值给 y）
    case [x, 0]:
        print(f"在X轴上，X坐标是 {x}")
    case [x, y]:
        print(f"普通坐标点: x={x}, y={y}")
    case _:
        print("不是坐标")
```

#### 匹配字典  

```python
user = {"name": "Alice", "age": 25, "city": "Beijing"}

match user:
    # 只要 name 是 Bob 就匹配，不管有没有 age
    case {"name": "Bob"}:
        print("你是 Bob")
    
    # 提取 name 和 age 的值
    case {"name": name, "age": age}:
        print(f"你好 {name}, 今年 {age} 岁")
        # 输出: 你好 Alice, 今年 25 岁
    
    case _:
        print("未知用户")
```

### 守卫  

#### 数值范围的守卫  

```python
point = [10, -5]

match point:
    # 1. 先匹配结构：是不是两个元素的列表？ -> 是，x=10, y=-5
    # 2. 再过守卫：x > 0 且 y > 0 吗？ -> 不是 (y是负数)
    case [x, y] if x > 0 and y > 0:
        print("第一象限")
        
    case [x, y]:
        print(f"其他象限或轴上: {x}, {y}")
```

#### 字典值的守卫  

```python
user = {"name": "小明", "age": 15}

match user:
    # 1. 匹配键：有 name 和 age 吗？ -> 有
    # 2. 守卫检查：age >= 18 吗？ -> 15 < 18，失败
    case {"name": name, "age": age} if age >= 18:
        print(f"欢迎 {name} 进入成人区")
        
    case {"name": name}:
        print(f"{name} 是未成年人，只能看动画片")
```

#### 类型守卫  

```python
data = {"id": "A-123", "value": 100}

match data:
    # 只有当 id 是整数时，才进入这个分支
    case {"id": int(my_id)}: 
        print(f"处理数字ID: {my_id}")
        
    # 只有当 id 是字符串时，才进入这个分支
    case {"id": str(my_id)}:
        print(f"处理字符串ID: {my_id}")
        # 输出: 处理字符串ID: A-123
```

## for  

它是“迭代器",有比较特殊的语法for...else...  

```python
for i in range(3):
    print(i)
else:
    print("循环正常结束，没有被 break 打断") # # 只有当循环跑完都没 break 时，才会执行这里

# 输出：
# 0
# 1
# 2
# 循环正常结束，没有被 break 打断

for _ in range(n)：# 没有用到参数时
    pass
```

## while  

while 循环与其他语言中的 while 循环在基本功能上相似，都是基于条件判断来控制循环。  

### 无限循环的惯用写法  

可用于游戏

```py
while True:
    user_input = input("请输入命令 (输入 'quit' 退出): ")
    if user_input == 'quit':
        break
    print(f"执行命令: {user_input}")
```

### while...else  

与for一样，else 块中的代码会在循环正常结束（即条件变为 False）后执行。但如果循环是被 break 语句强制中断的，else 块将不会执行

## round  

```py
# 1. 舍入到整数 (ndigits 省略或为 None)
round(3.6)        # 返回 4
round(3.4)        # 返回 3
round(2.5)        # 返回 2 (2是偶数，所以舍去)
round(3.5)        # 返回 4 (3是奇数，所以进位)
round(-0.5)       # 返回 0 (0是偶数)
round(1.5)        # 返回 2 (2是偶数)

# 2. 舍入到指定小数位
round(3.14159, 2) # 返回 3.14
round(2.675, 2)   # 返回 2.67 (注意：这并非错误，而是浮点数精度问题)
round(1.2345, 3)  # 返回 1.234

# 3. ndigits 为负数 (对小数点左侧进行舍入)
round(1234.56, -1)  # 返回 1230.0 (舍入到十位)
round(1234.56, -2)  # 返回 1200.0 (舍入到百位)
round(1234.56, -3)  # 返回 1000.0 (舍入到千位)
```

## \_\_name\_\_ 和 \_\_main\_\_  

这是一个非常重要的惯用法，用于判断一个模块是被直接运行，还是被其他程序导入。
当一个模块被直接运行时，它的 __name__ 变量值为 '__main__'。
当一个模块被导入时，它的 __name__ 变量值为模块的文件名（不含 .py）。
这个特性通常用于在模块中编写测试代码，确保这些代码只在模块自身运行时执行，而在被导入时不执行。

```python
def main():
    pass
# 只有当此文件被直接运行时，下面的代码才会执行

if __name__ == "__main__":
    main()
```

## try  

一个完整的 try 语句可以包含四个部分：try、except、else 和 finally。其中 try 是必需的，其他部分可以根据需要组合使用。  

```py
try:
    # 可能触发异常的代码块
    result = 10 / 0
except ZeroDivisionError:   # 如果这里不写异常类型，会捕获所有异常，有点危险
    # 处理 ZeroDivisionError 异常的代码块
    print("除数不能为零！")
except ArithmeticError: # 跟if语句一样，找到对应异常后，后面不执行，所以一定要子类在前面
    pass # ArithmeticError是上面的异常的父类，
else:
    # 只有当 try 块没有发生任何异常时，才会执行此代码块
    print("计算成功，结果是:", result)
finally:
    # 无论是否发生异常，此代码块都一定会执行
    # 如果异常，也会执行，否则try异常后，会暂停，可能会有风险
    print("无论结果如何，我都会执行。")
```

## 赋值

赋值本质都是给ID

| 维度 | 含义 | 对应运算符 | 判定标准 |
| :--- | :--- | :--- | :--- |
| ID (身份) | 内存地址 | `is` | 是不是同一个？ (最快) |
| Type (类型) | 对象种类 | `type()` | 是不是同类？ (前置过滤) |
| Value (值) | 数据内容 | `==` | 长得像不像？ (最慢) |

比较逻辑流
a is b：直接比 ID。ID 相同则 True。  
a == b：先看 Type（类型不同通常直接False），再看 Value（内容是否一致）。  

1. 不可变对象（整数、字符串、元组）  
修改 = 换 ID （看起来像给值）

2. 可变对象（list, dict, set）  
修改 = 保持 ID  

- 用is，如None，对象是否同一个
- ==，判断值，如，数值，字符串
