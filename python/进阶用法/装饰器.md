# 装饰器

允许在不修改原有函数或类代码的情况下，动态地为其添加新的功能。可以把它想象成一个“功能包装器”或“手机壳”：手机（原函数）的核心功能不变，但套上手机壳（装饰器）后，就获得了防摔、防滑等额外能力。  

装饰器本质上是一个高阶函数，它接收一个函数作为参数，并返回一个新的函数。

## 装饰器解释

```py
def welcome(fn):
    def wrapper(*args,**kwargs):
        print("Welcome")
        result = fn(*args,**kwargs)
        return result
    return wrapper

def my_fun(message: str):
    print(f"Hello {message}");

f1 = welcome(my_fun)
f1("Jack")  
# Welcome
# Jack

# 等价于
@welcome
def my_fun(message: str):
    print(f"Hello {message}");

my_fun("Jack")
```

上述的welcome就是装饰器，@welcome就能发挥作用

## 带wraps的装饰器

关于 @wraps(fn)，如果不使用它，装饰后的函数会丢失原始函数的 __name__、__doc__ 等元信息，变成 wrapper，这会给调试带来困扰。

```python
from functools import wraps
def welcome(fn):
    @wraps(fn) # 保留原函数的元信息
    def wrapper(*args,**kwargs):
        print('Welcome')
        return fn(*args,**kwargs)
    return wrapper
```

## 带参数的装饰器

在普通函数里套装饰器函数

```py
from functools import wraps
def welcome(name:str):  # name是参数
    def decorator(fn):
        @wraps(fn) # 保留原函数的元信息
        def wrapper(*args,**kwargs):
            print('Welcome')
            return fn(*args,**kwargs)
        return wrapper
    return decorator


@welcome("Tom")
def my_fun(*args,**kwargs):
    pass

my_fun(*args,**kwargs)

```

## 多个装饰器叠加

一个函数可以被多个装饰器装饰，它们的执行顺序是：装饰时从下到上。

```py
@decor1
@decor2
def target():
    pass

```
