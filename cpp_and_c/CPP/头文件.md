# 头文件

头文件的主要作用是声明，而不是定义。所以只放声明，不放定义  
因为如果头文件被多个 .cpp 文件包含，编译器会将头文件内容复制到每个 .cpp 中。如果里面有函数实现或变量定义，链接器在合并时会发现“多重定义”（Multiple Definition）错误。  

## 模板

```cpp
// 1. 包含防护 (Include Guard) 或 #pragma once
// 防止头文件被重复包含，导致重定义错误
#pragma once 

// 2. 必要的系统头文件
#include <vector>
#include <string>

// 3. 前向声明 (可选，用于减少编译依赖)
// 如果只用到指针或引用，可以先声明 class A; 而不需要 #include "A.h"
class DataProcessor; 

// 4. 命名空间 (可选，推荐)
namespace MyProject {

// 5. 常量定义 (推荐用 constexpr 代替 #define)
constexpr double kPi = 3.14159;
constexpr int kMaxRetries = 3;

// 6. 类或结构体定义
class Calculator {
public:
    // 构造函数
    Calculator();
    
    // 成员函数声明
    int add(int a, int b);
    
    // 内联函数 (短小的函数可以直接写在类里，隐含 inline)
    void reset() { sum_ = 0; }

private:
    int sum_;
};

// 7. 全局函数声明
void PrintVersion();

} // namespace MyProject

// 8. 模板实现 (必须在头文件中)
template <typename T>
T GetMax(T a, T b) {
    return (a > b) ? a : b;
}
```

## 对应的 .cpp文件

```cpp
#pragma once
#include <string>
#include <iostream>

class Student {
public:
    // 1. 构造函数声明
    Student(std::string name);
    
    // 2. 普通成员函数声明
    void study(const std::string& subject);
    
    // 3. 获取分数
    int getScore() const;

private:
    std::string m_name;
    int m_score;
};
```

```cpp
// 1. 必须包含对应的头文件
#include "Student.h" 

// 2. 构造函数的实现
// 格式：类名::函数名
Student::Student(std::string name) {
    this->m_name = name; // 具体赋值逻辑
    this->m_score = 0;   // 初始化分数
    std::cout << "学生 " << m_name << " 创建了！" << std::endl;
}

// 3. 普通函数的实现
void Student::study(const std::string& subject) {
    std::cout << m_name << " 正在努力学习 " << subject << "..." << std::endl;
    m_score += 10; // 学习后分数增加
}

// 4. 常量函数的实现 (记得加 const)
int Student::getScore() const {
    return m_score;
}
```

## 错误用法

| 错误做法 | 后果 | 正确做法 |
| :--- | :--- | :--- |
| `using namespace std;` | 污染全局命名空间，导致名字冲突 | 在头文件中写全称 `std::vector`，在 `.cpp` 中才用 `using` |
| `int g_val = 10;` (无 inline) | 链接报错 "multiple definition" | `extern int g_val;` (声明) 或 `inline int g_val = 10;` |
| 在头文件中写复杂函数体 | 增加编译时间，违反封装 | 在头文件声明，在 `.cpp` 实现 |
| 忘记加 `#pragma once` | 重复定义错误 | 总是加上防护 |

### 包含防护

有两种写法，推荐使用 #pragma once，因为它更简单且现代编译器都支持。  
现代写法： `#pragma once`  
传统写法（宏）：

```cpp
# ifndef MY_HEADER_H
# define MY_HEADER_H
// 内容
# endif
```

### inline

使用 inline 变量。这允许变量在多个文件中定义，但链接器会合并它们。不再需要.cpp文件定义了；如果是简单的函数，想把函数实现写在头文件里，必须加 inline。此外即使不写在头文件里，对于简单函数+inline会更快，因为是把简单函数的函数体粘贴，不像一般函数是非常复杂的过程。而且此方法比宏好。  

```cpp
// config.h
inline int g_config_value = 100; // 安全，推荐
```

#### 旧标准： 使用 extern 声明

如果想让多个文件共用同一个全局变量,遵循“一处定义，多处声明”的原则。  

- 定义：真正分配内存的地方（只能有一次）。
- 声明：告诉编译器这个变量存在，类型是什么（可以有多次，通常加 extern）。

```cpp
// config.h
extern int g_config_value; // 只是声明，不分配内存

// config.cpp
int g_config_value = 100;  // 在这里真正定义
```
