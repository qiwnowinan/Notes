# 类与对象

## 构造与析构

- 构造：无返回值，与类名一致，在构造对象时，自动调用  
- 析构：在类名前加个~,无参数，对象销毁时，自动调用  
析构顺序与构造相反  

```cpp
class Tree{
    int height;
public:
    Tree(int height);
    ~Tree();
};

Tree t(12);
```

## 访问权限

- public 任何人可以访问
- private 只有这个类可以访问
- protected 只有这个类和它的子类可以访问

## 成员初始化列表

此方法效率更高  
不过依然是看类定义的顺序初始化  

```cpp
class Point {
private:
    const float x, y; // 注意：这里 x 和 y 是 const（常量）

    // 构造函数
    Point(float xa = 0.0, float ya = 0.0)
    : y(ya), x(xa)  // <--- 这里就是“初始化列表”
    {}              // 构造函数体为空
};
```

## 继承

不继承构造和析构

```cpp
#include <iostream>
using namespace std;

class A{
public:
    A():i(0){ cout << "A::A()" << endl;}
    ~A(){ cout << "A::~A()" << endl;}
    void print(){ cout << i << endl;}
protected:
    void set(int ii){ i = ii; }
private:
    int i;
};

class B :public A{ // public是必须的
public:
    void f(){ set(20); print(); i = 30;/*这是错的，i存在，可以使用，无法修改*/}
};

int main(){
    B b;
    b.set(10);//错误，main里无法用protected
    b.print();
}
```

## 引用

```cpp
char c;
char *p = &c;
char& r = c;// r是c的别名，必须初始化,一旦初始化，就不能改变指向，即不能char& r = d;.

const int& z = x;//z 是 x 的别名，但是通过 z 这个别名，只能读取 x，不能修改 x。

void f(int& x);
f(y);//给的就是y,在函数里改 x，实际上就是在改外面的 y。
f(i * 3); //错误

## virtual和多态

如果没有 virtual，C++ 是静态绑定（早绑定）的，编译器只看指针的类型，不看指针实际指向的对象。
如果有一个virtual,析构必须是virtual.

```cpp
class Animal {
public:
    void speak() { cout << "动物叫" << endl; }
};

class Dog : public Animal {
public:
    void speak() { cout << "汪汪汪" << endl; }
};

int main() {
    Animal* ptr = new Dog(); // 基类指针指向子类对象
    ptr->speak();            // 输出："动物叫"
    // ❌ 悲剧：虽然实际是狗，但因为指针是 Animal*，编译器只调用了 Animal 的 speak
}

class Animal {
public:
    virtual void speak() { cout << "动物叫" << endl; } // ✅ 加上 virtual
};

class Dog : public Animal {
public:
    void speak() override { cout << "汪汪汪" << endl; } // ✅ 自动变成虚函数
};

int main() {
    Animal* ptr = new Dog();
    ptr->speak();            // 输出："汪汪汪"
    // ✅ 成功：程序在运行时发现 ptr 实际指向的是 Dog 对象，调用了 Dog 的 speak
    delete ptr;
}
